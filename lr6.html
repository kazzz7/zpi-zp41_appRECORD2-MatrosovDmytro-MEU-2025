ТЕМА:

АСИНХРОННІСТЬ. ПРОМІСИ. HTTP-ЗАПИТИ. REST API. AJAX. КРОС-ДОМЕННІ ЗАПИТИ.

МЕТА:

Придбати практичні навички роботи зі створення промісів, методів then(), catch(), finally(); ланцюжків промісів; роботи з протоколами HTTP/HTTPS, REST API та Fetch API для AJAX-запитів до публічного API сервісу.

ПОСИЛАННЯ:

  • Репозиторій звітного HTML-документа
  • Жива сторінка звітного HTML-документа

ЗАВДАННЯ №1: Таймер зворотного відліку

  • Підключені бібліотеки flatpickr та iziToast
  • При першому завантаженні сторінки кнопка Start неактивна
  • При кліку на інпут відкривається календар для вибору дати
  • Якщо обрана дата з минулого, з’являється повідомлення “Please choose a date in the future” та Start залишається неактивною
  • При обранні дати з майбутнього кнопка Start стає активною
  • Після натискання Start інпут і кнопка блокуються, на сторінці відображається зворотний відлік у форматі xx:xx:xx:xx, таймер оновлюється щосекунди і зупиняється при досягненні нуля

ЗАВДАННЯ №2: Генератор промісів

  • Розмітка форми з полем delay (ms), двома радіокнопками state=fulfilled/rejected та кнопкою Submit
  • Після сабміту створюється проміс із заданою затримкою
  • У залежності від обраного state показується iziToast-сповіщення з повідомленням про успіх або відмову та зазначеним delay

ЗАВДАННЯ №3: Пошук зображень по Pixabay API

  • Форма з полем query для пошуку та кнопкою Search
  • Запит до Pixabay з вашим API-ключем, параметрами image_type=photo, orientation=horizontal, per_page=12, safesearch=true
  • При отриманні результатів галерея рендериться динамічно, перед кожним новим пошуком старі елементи очищаються
  • Для перегляду збільшених зображень використано SimpleLightbox із викликом refresh() після оновлення
  • В разі помилки або порожнього результату показується iziToast-повідомлення
